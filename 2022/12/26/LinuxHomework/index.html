朱嘉美的Linux大作业

一、 Linux下安装git

使用git管理大作业相关的代码、配置文件等，要求有能反映大作业过程的git提交记录。

1、下载git，直接 sudo apt-get(或yum) install git ，下载完后查看git版本。

2、配置用户名和用户邮件

3、配置Git的私钥和公钥.密码为空

4、上github打开设置，进入个人设置点击 new ssh key，inux上打开id_rsa.pub文件，复制密钥到github上

5、测试是否连接（如图，已经连接上了）

6、github创建项目，将远程仓库与本地仓库连接起来

二、Linux下，运行系统监控

1、查看CPU、内存、进程监控

TOP命令是Linux下一个实时的、交互式的，对操作系统整体监控的命令，可以对CPU、内存、进程监控。是Linux下最常见的监控命令。

2、查看内存监控

free 命令可以查看当前系统内容的使用情况:free -m: 以MB为单位显示系统内存的使用情况，同理，也可以使用 -k， -g等其他的单位显示
free 命令从两个诶度统计了内存的使用情况
1、第一行Mem：从操作系统角度统计了内存的total、used、free、buffers、cached、available
2、第二行 Swap: 从应用程序角度统计了内存的total、used、free、buffers、cached、available

3、查看当前系统磁盘空间的使用情况

df命令可以查看当前系统磁盘空间的使用情况

4、查看网络监控

Netstat 命令用于显示各种网络相关信息，如网络连接，路由表，接口状态 (Interface Statistics)，masquerade 连接，多播成员 (Multicast Memberships) 等等。

从整体上看，netstat 的输出结果可以分为两个部分:一个是 Active Internet connections，称为有源 TCP 连接，其中”Recv-Q”和”Send-Q”指%0A的是接收队列和发送队列。这些数字一般都应该是 0。另一个是 Active UNIX domain sockets，称为有源 Unix 域套接口(和网络套接字一样，但是只能用于本机通信，性能可以提高一倍)。

5、查看磁盘IO

iostat 主要用于输出磁盘IO 和 CPU的统计信息，用来动态监视系统的磁盘操作活动。iostat程序需要通过sysstat包安装。

iostat -x -k 1

  • -x：展示磁盘的扩展信息
  • -k：以k为单位展示磁盘数据
  • 1：没1秒刷新一次数据

三、Hexo的搭建

配置好Hexo相关环境

Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Coding上，是搭建博客的首选框架。

Hexo搭建步骤

  1. 安装Git(因为前面已安装，所以查看下版本号即可。如果没有安装那就先需安装)

  2. 安装Node.js，Hexo是基于nodeJS编写的，所以需要安装一下nodeJs和里面的npm工具。安装完后查看是否安装成功。

  3. 安装Hexo

    新建完成后，打开hexo的服务，在浏览器里输入 xxx:4000(xxx为你服务器的公网ip)，按理说可以看到你生成的博客。但有时会因为端口被占用或者服务器的4000端口没有对外开放，在服务器上成功安装了hexo，且运行起来了，但是始终无法访问4000端口。

    如果出现下图的页面，则表示ok了。
